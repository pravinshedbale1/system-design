# System Design Mastery

A comprehensive, storytelling-driven system design learning path designed to crack any system design interview at any company in the world.

## 🎯 Goal

Walk into any system design interview — Google, Amazon, Meta, Netflix, Uber — and design any system from scratch with confidence, depth, and storytelling.

## 🧭 How to Use This Repository

1. **Start here**: Read `00_MASTER_PLAN.md` for the full journey
2. **Track progress**: `CONVERSATION_STATE.md` is updated after every session
3. **Learn concepts**: Each pillar in `concept_knowledge/` is populated through conversations
4. **Design systems**: Each system gets a detailed doc in `designs/`
5. **Practice recall**: Use `flashcards/` and `story_bank/` before interviews
6. **Review**: Follow `spaced_repetition/review_schedule.md`

## 📅 Timeline

- **Phase 1** (Weeks 1-4): Building Blocks — master the fundamentals
- **Phase 2** (Weeks 5-10): Classic Systems — design the top 15 systems
- **Phase 3** (Weeks 11-16): Advanced + Mock Interviews — handle anything

## 🎭 Philosophy

> **A great system design answer is a STORY, not a diagram.**
> Every concept is learned through analogy. Every system is a journey from simple to complex.
> The goal isn't to memorize 20 designs — it's to understand 8 pillars so deeply 
> that you can design ANYTHING.
